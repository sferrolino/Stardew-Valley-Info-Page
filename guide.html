Keep it Simple!

A website that is too cluttered with information or components is difficult for a user to stay focused. Any information that the webpage is trying to convey could easily be lost. We tried our best to keep our webpage as simple and clean looking as possible so that the information we want to get accross is clear and easy to understand.





Backend Framework: Laravel

There are not too many things on the backend that can help your webpage look better on the front end. However, there are a few things that you can configure on Laravel that could make the user's experience during their visit a little better.





Turn off Error Details

After installing Laravel, error details are enabled by default for your application. If an error occurs, the user will be shown a detailed stack trace and error message. Make sure to turn these details off in the your_project/config/app.php under debug option to false.

.




Customize Error Pages

Laravel allows you to create custom HTTP error pages. A common error that you could consider creating a custom page for, would be the "404 not found" error. This can be done easily by creating a 404.blade.php file inside the directory resources/views/errors. Many websites now choose to customize their error pages in a fun way.
